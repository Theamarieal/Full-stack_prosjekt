Module: router

Vue Router configuration for the Checkd application. All protected routes require authentication via the `requiresAuth` meta field. Role-based access control is enforced through the `roles` meta field, which specifies which user roles are permitted to access a given route. Available roles: 'EMPLOYEE', 'MANAGER', 'ADMIN' Route access summary: - Public: /login, /register - All authenticated: /, /temperature, /temperature-history, /training, /checklists, /alcohol, /deviations, /deviations/new - Manager + Admin: /reports, /manage-checklists - Admin only: /admin A global navigation guard enforces authentication and role checks before each route transition. Unauthenticated users are redirected to /login, and users without the required role are redirected to the dashboard.
